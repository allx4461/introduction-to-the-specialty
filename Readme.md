# Microservice Ascii Aquarium & Chat

Интерактивный ASCII-аквариум и анонимный чат сгорающих сообщений, построенный на микросервисной архитектуре.

---

## Схема архитектуры системы

Ниже представлена схема взаимодействия компонентов приложения при запуске через Docker Compose:

```mermaid
graph TD
    User([Браузер пользователя]) -->|1. Запрос HTML/JS (порт 80)| Frontend[Контейнер Nginx: Frontend]
    User -->|2. Анимация аквариума (порт 8080)| BackendCPP[Контейнер C++: Bulbul]
    User -->|3. Оправдания и чат (порт 5000)| BackendPy[Контейнер Python: Informer]
    BackendPy -->|4. Внешний запрос| ExternalAPI[Внешнее API оправданий]

    subgraph Docker Compose Network
        Frontend
        BackendCPP
        BackendPy
    end
```

## Инструкция по запуску

Вы можете запустить проект двумя способами: локально в контейнерах или вручную.

### Способ 1. Запуск через Docker Compose (Рекомендуется)

Для сборки и запуска всех микросервисов в один клик выполните в корне проекта:
```bash
docker compose up --build
```

После успешного запуска:
* **Фронтенд (UI):** Откройте в браузере [http://localhost](http://localhost) (порт 80).
* **C++ Сервер (Аквариум):** Доступен на `http://localhost:8080`.
* **Python Сервер (Чат и информер):** Доступен на `http://localhost:5000`.

### Способ 2. Запуск на удаленном сервере (VPS) (для обмена соо)

Для развертывания проекта в режиме совместного доступа для нескольких пользователей:
1. Клонируйте репозиторий на ваш сервер.
2. Перейдите на ветку с чатом: `git checkout chat-feature`.
3. Запустите сборку контейнеров:
   ```bash
   docker compose up --build -d
   ```
   *(Флаг `-d` запустит контейнеры в фоновом режиме).*
4. Откройте в браузере `http://IP_ВАШЕГО_СЕРВЕРА` (например, `http://185.189.14.184`). Фронтенд автоматически определит IP-адрес сервера и подключит вас к общему чату и аквариуму.

## траблшутинг

### Ошибка: `port is already allocated` или `address already in use`
Эта ошибка означает, что порты `80`, `8080` или `5000` уже заняты другими процессами на вашем компьютере.

**Решение:**
1. Найдите и остановите процессы, занимающие порты.
   * На Linux/macOS проверить занятые порты можно командами:
     ```bash
     sudo lsof -i :80
     sudo lsof -i :8080
     sudo lsof -i :5000
     ```
   * Убить процесс можно по его PID: `kill -9 <PID>`.
2. Либо измените внешние порты в файле `docker-compose.yml` (левая часть в секции `ports`, например, изменить `"80:80"` на `"8081:80"`).
