# Microservice Ascii Aquarium

## идея
интерактивный аски-аквариум, можно добавлять рыбов. можно читать оправдания, которые обновляются каждые 10 сек


## функциональность
* **Рендеринг аквариума:** Непрерывная генерация и анимация ASCII-графики с движущимися объектами (рыбы, водоросли).
* **Интерактивность:** Возможность динамически добавлять рыб в аквариум через UI (кнопки Fish 1, Fish 2, Fish 3).
* **Генератор оправданий:** Интеграция со сторонним API для получения отговорок, обновляется каждые 10 секунд. (получено из [отсюда](https://github.com/public-apis/public-apis#dictionaries))
 
## архитектура
1. **Frontend :** HTML/JS страница. Отвечает за опрос бэкендов (long-polling/setInterval) и обновление DOM-дерева (`innerHTML` для отрисовки графики).
2. **Backend 1 :** C++ сервер. Отвечает за логику координат, анимацию и отдачу "кадров" аквариума.
3. **Backend 2 :** Python (FastAPI) сервис. Выступает в роли информера, который ходит во внешний интернет за данными, форматирует их и отдает клиенту

## Инструкция по запуску

Вы можете запустить проект двумя способами: локально вручную или в один клик через Docker Compose.

### Способ 1. Запуск через Docker Compose (рекомендуется)
Для запуска всех компонентов системы в контейнерах выполните в корне проекта команду:
```bash
docker compose up --build
```
После запуска:
* Фронтенд будет доступен по адресу: `http://localhost`.
* C++ Сервер аквариума будет доступен по адресу `http://localhost:8080`.
* Python Сервис оправданий будет доступен по адресу `http://localhost:5000`.


### Способ 2. Локальный запуск вручную

#### 1. Запуск Python бэкенда (Informer)
Установите зависимости и запустите FastAPI-сервер:
```bash
cd informer
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 5000
```

#### 2. Компиляция и запуск C++ бэкенда (Bulbul)
Соберите C++ код и запустите исполняемый файл:
```bash
cd bulbul
g++ -std=c++17 src/*.cpp -o aquarium_app -pthread
./aquarium_app
```

#### 3. Запуск фронтенда
Откройте файл `web/index.html` в любом браузере. Фронтенд автоматически настроен на поиск бэкендов на `localhost`.

## Тестирование и CI
* В репозитории настроен автоматический пайплайн CI (GitHub Actions) в файле `.github/workflows/config.yml`.
* Пайплайн запускается при создании Pull Request в ветку `main`, форматирует и линтит Python-код, прогоняет тесты и проверяет компилируемость C++ бэкенда.
* Запуск тестов локально:
  ```bash
  pytest informer/
  ```
